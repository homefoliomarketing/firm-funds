# Request for Quote: Commission Advance Web Portal

## Overview

I need a fixed-price quote to build a web-based commission advance portal from scratch. The app is for a Canadian company (Ontario) that purchases pending real estate commissions from agents at a discount. This is NOT a loan product — it's a commission purchase agreement (important for all user-facing language).

The attached requirements document (Firm_Funds_Current_Build_Requirements.docx) has the full specification — database schema, every page, every feature, every security requirement. **Read it fully before quoting.** This message summarizes the scope so you can quickly assess fit.

---

## What the App Does

Three user types interact through role-specific dashboards:

1. **Real estate agents** submit advance requests, upload supporting documents, and track their deals
2. **Partner brokerages** view their agents' deals, earn referral fees, and flag/unflag agents
3. **Admin (me)** underwrites deals via a 17-item checklist, approves/denies/funds deals, manages the full deal lifecycle

Every deal flows through 8 statuses: Submitted → Under Review → Approved → Funded → Repaid → Closed (plus Denied and Cancelled as terminal states). Status transitions are enforced server-side.

## Scope Summary — What Must Be Built

### Pages/Views (8 total)
1. **Login page** — email/password, role-based redirect, password reset, error handling
2. **Admin dashboard** — stats cards (4), search bar, status filter tabs, paginated deals table (15/page), optimized parallel queries
3. **Admin deal detail** — full financial breakdown, visual status pipeline, 17-item underwriting checklist (4 categories, toggleable checkboxes with progress bar), document list with download/delete, status action buttons with denial modal, deal notes
4. **Agent dashboard** — stats cards (3), deals list, "new advance request" button
5. **Agent deal submission form** — 5 form fields with live calculator preview (debounced server call with race condition protection), confirmation modal, Zod validation
6. **Agent deal detail** — financial breakdown, document upload (10 document types, file validation: type/size/extension), document list with signed URL downloads, edit mode when deal is in "submitted" status with auto-recalculation
7. **Brokerage dashboard** — stats cards (3), two-tab layout (deals with expandable rows + agents with flag/unflag toggle), pagination

### Shared Components (2)
8. **Theme toggle** — sun/moon button for dark/light mode switching
9. **Session timeout** — inactivity auto-logout (15 min admin, 30 min agents), warning banner, activity tracking

### Authentication & Authorization
- Supabase Auth with email/password
- Server-side JWT validation via middleware on every request
- Role-based route protection (4 roles: super_admin, firm_funds_admin, brokerage_admin, agent)
- Row Level Security (RLS) on ALL database tables with 4 helper functions
- Agents see only their own data; brokerages see only their brokerage's data; admins see everything

### Database (7 tables + 1 trigger)
- **user_profiles** — links auth users to roles
- **brokerages** — partner brokerages with configurable referral fee %
- **agents** — linked to brokerages, with flag and outstanding recovery fields
- **deals** — 20+ columns including all financial calculations, status, EFT tracking (JSONB)
- **deal_documents** — file metadata (10 document types), actual files in Supabase Storage
- **underwriting_checklist** — 17 items auto-generated by DB trigger on deal creation
- **audit_log** — immutable append-only audit trail
- **notification_log** — table structure only (not wired to email yet)
- Comprehensive RLS policies on every table
- Supabase Storage bucket with upload policies (10MB limit, file type whitelist)

### Financial Calculation Engine
- 7 formulas using integer-cents rounding (no floating-point errors)
- Centralized in one module, imported everywhere
- Recalculation on funding (uses actual days from funding date to closing, not original submission date)
- Live preview in agent form (debounced 400ms with race condition protection)
- Discount rate: $0.75 per $1,000 of net commission per day until closing
- Brokerage referral fee: configurable per brokerage (default 20%)

### Server Actions (8 total)
All mutations go through server actions with: authentication → Zod validation → execution → audit logging → result return. Includes optimistic locking on status updates.

### Dark/Light Theme System
- ~40 color tokens in a React Context provider
- Dark mode default (charcoal grey #121212, not pure black)
- Light mode (cream #F5F3EF base)
- localStorage persistence
- Brand colors: Charcoal (#1E1E1E), Sand/Gold (#C4B098, #A8926F)
- 8 distinct status badge styles

### Security (All Required)
- Server-side JWT validation on every request
- RLS on all tables
- Zod validation on all inputs
- File upload validation (MIME type, extension, 10MB limit)
- Filename sanitization (timestamp + UUID)
- Audit logging on all significant actions
- Optimistic locking on status updates
- Session timeout with role-based durations
- Security headers: HSTS, CSP, X-Frame-Options DENY, X-Content-Type-Options, Referrer-Policy, Permissions-Policy
