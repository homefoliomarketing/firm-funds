## Prompt (paste this)

Make the brokerage agent roster importer accept `.xlsx` files, not just `.csv`. Right now an
admin can only bulk-import agents from a CSV. We also hand new brokerages a nicely designed
`.xlsx` onboarding roster (`Firm-Funds-Agent-Onboarding-Roster.xlsx`), and there is already a
`public/firm-funds-agent-import-template.xlsx` template, but neither can actually be uploaded
because the importer rejects everything that is not `.csv`. Add `.xlsx` support end to end so a
filled-in roster imports in one step.

Requirements:

1. **Parse xlsx on the server.** The import runs in a `'use server'` action, so the parser must
   work server-side on Netlify. There is currently **no** spreadsheet library in `package.json`
   (verify with `grep -iE "xlsx|exceljs|sheetjs" package.json`). Add one (SheetJS `xlsx` or
   `exceljs`) and confirm it bundles cleanly under Next.js 16 + Turbopack and Netlify's stricter
   TS. Read `node_modules/next/dist/docs/` before writing code, per AGENTS.md.

2. **Branch on file type, reuse the existing pipeline.** In `bulkImportAgentsCsv`
   (`lib/actions/admin-actions.ts`, ~line 799):
   - Currently: `if (!file.name.toLowerCase().endsWith('.csv'))` rejects the file (~line 807),
     then `rowsToBulkAgents(parseCsv(await file.text()))` (~line 814).
   - Change so `.xlsx` (and probably `.xls`) is also accepted. For xlsx, read
     `await file.arrayBuffer()`, parse the **first worksheet** into the same shape `parseCsv`
     returns (an array of string-arrays, one per row), then feed it through the **existing**
     `rowsToBulkAgents` helper. Do NOT duplicate the header-detection / column-mapping logic.
   - The function name `bulkImportAgentsCsv` becomes a misnomer; consider renaming to
     `bulkImportAgentsRoster` and updating the import in
     `app/(dashboard)/admin/brokerages/page.tsx` (it is imported at the top and called at
     ~line 879 and ~line 1185).

3. **Header row can be deeper than row 10.** `rowsToBulkAgents` (~line 744) only scans the
   **first 10 rows** for a header row containing "first" and "email". Our branded roster
   (`Firm-Funds-Agent-Onboarding-Roster.xlsx`) puts its header on **row 12** (title banner +
   "Brokerage Details" block sit above it). Either widen the scan (e.g. first 25 rows) or scan
   all rows. The roster also has extra columns the mapper will simply ignore (`#`, `Status`,
   `Notes`) because `findValue` (~line 776) only maps known headers (first/last/email/phone/
   reco/address) - that is fine, leave it.

4. **Keep the security + limit guards.** The CSV path already blocks formula injection via
   `hasSpreadsheetFormulaRisk` (~line 732) and enforces `MAX_ROSTER_ROWS = 200` (~line 678) and
   `MAX_ROSTER_CSV_BYTES = 256 * 1024` (~line 677). Apply the SAME formula-risk check to xlsx
   cell values, keep the 200-row cap, and set a sane byte cap for xlsx (binary, so 256KB is
   tight - pick a bounded limit and guard against zip-bomb / huge sheets; the 200-row cap helps).

5. **Update the upload UI.** Two file inputs use `accept=".csv,text/csv"` in
   `app/(dashboard)/admin/brokerages/page.tsx` (~line 1766 create-form roster upload, and
   ~line 2252 per-brokerage import). Add `.xlsx` (and the xlsx MIME
   `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`). Update the helper text
   at ~line 1756 ("Upload a .csv with columns: ...") to say csv **or** xlsx.

6. **Verify for real.** Type-check (`npx tsc --noEmit`, ignore `.next/`), run `npx vitest run`
   (103 tests pass today - keep them green), then log into the admin portal
   (`bud@firmfunds.ca` / see memory `reference_admin_password.md`) and actually upload
   `Firm-Funds-Agent-Onboarding-Roster.xlsx` (or a fresh copy) into a test brokerage. Confirm all
   47 roster rows import with the right names, and that the `#` / Status / Notes columns are
   ignored without error. Screenshot the result.

7. **Update docs in the same commit** (per CLAUDE.md / CONTRIBUTING.md): the import flow, the
   `.csv`-only mention in any help article under `content/help/`, and note xlsx support wherever
   roster import is described. Refresh the relevant `_Last updated:_` lines.

Then push to main (confirm with Bud first, per CLAUDE.md).

## Why this is needed (context)

- 2026-06-09: Built a branded `.xlsx` onboarding roster for Bud to send new brokerages
  (`Firm-Funds-Agent-Onboarding-Roster.xlsx`, generated by `scripts/build-onboarding-roster.py`).
  Discovered the app's importer only accepts CSV, so a filled-in roster cannot be imported in one
  step. This task closes that gap.
- The importer is a server action, not a REST route: `bulkImportAgentsCsv` in
  `lib/actions/admin-actions.ts`. Helpers: `parseCsv`, `rowsToBulkAgents`, `findValue`,
  `hasSpreadsheetFormulaRisk`, `normalizeHeader` in the same file.
